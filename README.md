# 老板.skill

> 你花了三年时间研究老板喜欢喝什么咖啡、什么时候不能找他说事、他那句"你来 decide"到底是放权还是埋坑——
>
> 现在，把这些心得蒸馏成一个 AI Skill，永久复用。

---

你有没有经历过以下场景：

- 精心准备了一小时的方案，他看了一眼说"逻辑跑通了吗" —— 然后你回去又改了三天
- 会议室里突然被问"这个数据怎么来的"，当场社死
- 发了条消息等了两小时，他回了俩字"知道了" —— 然后你完全不知道他的意思是同意还是已读不理
- 好不容易找到他申请资源，他说"现有的人为什么不够" —— 你没准备这个问题

**老板.skill 解决的就是这个问题。**

把你老板的决策逻辑、沟通偏好、拍板规律、已知雷区，蒸馏成一个 AI Skill。
汇报前过一遍，避雷检查，把你想说的话翻译成他喜欢听的语言。

从此，1:1 不再靠运气。


## 用法

```
/create-boss
```

回答 4 个问题：老板叫什么、他是什么人、你们什么关系、你对他的核心印象。
原材料可以是飞书消息、会议纪要、他写的文章，甚至就是你的记忆。

全部可以跳过，仅凭你的描述也能生成。但原材料越多，Skill 越准。

生成后，用 `/{slug}` 召唤他的灵魂。

## 它能帮你做什么

**汇报前**："帮我用 L总 的视角 review 一下这个方案，哪里会被问倒？"

**申请资源前**："我要找 L总 申请 2 个 HC，他会问什么？我怎么回答？"

**被推回之后**："他说'逻辑跑通了吗'，我应该改什么？"

**开口前**："帮我把这句话翻译成 L总 喜欢听的方式说"

**踩雷之后**："我刚才在会议上说了什么让他脸色变了？"

---

## 生成的 Skill 长什么样

每个老板 Skill 包含两部分：

**Part A — 汇报指南**：他喜欢什么格式、什么结构、什么时候不要去找他、哪些话题是雷区、他"知道了"到底是什么意思。

**Part B — 人格画像**：5 层结构，从硬规则到人际行为，覆盖他的决策触发器、表达风格解读、以及一张"他的语言翻译表"——把你的人话翻译成他的人话。

运行逻辑：`你的问题 → Skill 预判他的反应 → 给你可执行的建议`

---

## 支持的标签

给你的老板贴几个标签，Skill 会自动翻译成具体的行为规则：

**管理风格**：PPT 控 · 数据驱动 · 微操狂魔 · 放权型 · 结果导向 · 只问结论不问过程

**企业文化**：字节范 · 阿里味 · 腾讯味 · 华为味 · 创业公司派 · 500 强流水线

**性格**：话少但有话必中 · 会议前必须对齐 · 最怕没有结论 · 特别在意竞品 · 情绪化 · 向上管理高手 · 喜欢带节奏

**职级**：字节 2-1~VP · 阿里 P8~P11 · 腾讯 T3~T4 · 百度 T8~T9 · 美团 P7~P8 · 各类创业公司 CXO

## 效果示例

```
你        ❯ 我今天要向 L总 申请再招 2 个前端，帮我准备一下

老板.skill ❯ 先说个坏消息：L总 的第一反应大概率是
             "现有的人为什么不够"，这个问题你没有答案就别去。

             你需要提前装进口袋的三件东西：

             ① 当前团队的时间分配表（截图最好）
                → 让他看到，不是人不够努力，是事真的多

             ② 2 个 HC 的 ROI 测算
                → "招进来 3 个月后，DAU 预期提升 X%"
                → 他是数据驱动型，没数字等于没方案

             ③ Plan B 放在兜里
                → 如果他不批，你打算裁哪个需求
                → 主动说出来，他会觉得你成熟

             开场白：
             "L总，我想汇报一下前端现状，我有个具体申请，数据在这里……"

             千万别说：
             "我感觉人手不够，想再招几个。"
             （"感觉"二字会让他停止思考你后面说的所有话）
```

### `/boss-radar` — 老板活跃度感知

根据老板的行为规律，生成他今天的"干扰指数预报"。

```
你        ❯ /boss-radar 今天

老板.skill ❯ 📊 今日 L总 活跃度预测：

             🟢 09:00–10:00  他在赶周报，不会找你
             🔴 10:00–11:30  全员周会，精神高度集中，不适合划水
             🟡 11:30–13:00  午饭+休息，状态模糊，视情况
             🟢 13:00–15:00  ⭐ 黄金时段 ⭐ 他通常在看文档不发消息
             🔴 15:00–17:00  下午容易开临时会，保持基本在线
             🟢 17:30 之后   他开始准备明天的事，基本不找人

             今日推荐深度专注窗口：13:00–15:00（2小时）
             建议在此期间处理：需要大块时间的个人成长型任务 📚
```

### `/meeting-audit` — 会议必要性鉴定

把会议邀请丢进来，Skill 帮你判断这个会你到底需要几成出力。

```
你        ❯ /meeting-audit "季度跨部门对齐会，议程：各组同步进展"

老板.skill ❯ 🔍 会议价值评估报告

             会议类型：信息同步型（非决策）
             L总 关注度：中（他会扫一眼有没有他的 team 的问题）
             你的发言机会：约 3 分钟

             建议出力等级：★★☆☆☆

             推荐策略：
             • 提前准备 1 张进展截图，到你发言时共享屏幕
             • 其余时间可处理消息、整理文档（摄像头保持开启即可）
             • 结束前 5 分钟恢复注意力，以防点名

             风险提示：L总 有 23% 概率临时提问，
             建议备好一句："我们进展顺利，细节我会后续同步给你。"
```

### `/task-roi` — 任务价值评估器

这件事老板到底在不在乎？帮你决定投入几成力气。

```
你        ❯ /task-roi 整理竞品分析报告，下周例会用

老板.skill ❯ 📋 任务 ROI 分析

             L总 对竞品的关注度：🔥 极高（他自己维护竞品 tracker）
             这个报告出现在例会的概率：85%
             他会细看的概率：70%
             他会当场提问的概率：60%

             建议投入等级：★★★★★ 认真做

             重点提示：
             • 一定要有数据对比，不能只截图
             • 结尾加一句"我们的差异化机会在于 X"
             • 控制在 5 页以内，他不看长文档

             ⚠️ 这个不建议省力，踩到他的关注点了。
```

```
你        ❯ /task-roi 更新内部 wiki 的团队介绍页

老板.skill ❯ 📋 任务 ROI 分析

             L总 上次看内部 wiki 的时间：未知（大概率从没看过）
             这件事影响你绩效的概率：3%
             做完会有人夸你的概率：2%

             建议投入等级：★☆☆☆☆ 差不多就行

             推荐策略：
             • 复制上个季度的，改一下日期和人数
             • 预计耗时：8 分钟
             • 省下来的时间建议用于：午睡、摸鱼、或上面那份竞品报告
```

### `/weekly-inflate` — 周报充实器

把你这周真实产出丢进来，Skill 帮你翻译成一份看起来分量十足的周报。

```
你        ❯ /weekly-inflate
             这周干了啥：
             - 开了好几个会
             - 改了个 bug
             - 看了竞品

老板.skill ❯ ✍️ 充实版周报（L总 口味）

             【本周重点进展】

             ▶ 跨部门协同推进（3 项）
               参与产品/研发/数据多轮对齐会议，推动 X 议题达成共识，
               确保 Q4 目标路径清晰。

             ▶ 系统稳定性优化
               定位并修复影响用户体验的关键 bug（涉及 XX 模块），
               已验证上线，预计降低相关投诉率。

             ▶ 竞品动态追踪
               完成本周竞品调研，发现竞品在 X 方向有新动作，
               已整理关键信息，下周例会同步。

             【下周计划】
             继续推进 XX，预计产出 XX。

             ---
             字数：128 字 / 原始素材：27 字
             膨胀系数：4.7x ✨
             L总 满意度预测：🟢 通过
```

### `/invisible-cloak` — 低存在感模式指南

当你需要一段时间"消失"专注做自己的事，Skill 告诉你怎么保持合理的在线感。

```
你        ❯ /invisible-cloak 我需要今天下午 3 小时不被打扰

老板.skill ❯ 🫥 低存在感方案（今日适用）

             准备工作（5 分钟）：
             ① 在群里发一条消息，主动同步一个小进展
               → 让他觉得你活跃，后续不太会主动找你
             ② 飞书状态改为"专注中"或"外出"
               → 合理挡掉非紧急消息

             执行期间：
             • 每 45 分钟回一次消息（不要完全消失，偶尔冒泡）
             • 如果他发消息：秒回"好的，我处理完手头这个马上"
               → 争取 15 分钟缓冲，足够你切换状态

             收尾（10 分钟前）：
             • 恢复正常状态，看一下有没有漏掉的 @ 消息
             • 准备好一句今天的"小结论"，万一被问起有得说

             今日风险等级：🟢 低（他下午有外部会议）
             预计可用时间：2.5 小时（留 30 分钟缓冲）

             祝深度专注愉快 🎧
```

## 注意事项

- 原材料质量决定 Skill 精度：**他主动写的长文 > 他的决策回复 > 日常消息 > 你的主观描述**
- 建议优先收集：会议纪要里他的发言 · 他群发的通知 · 他写的复盘文章
- 用起来发现哪里不对？直接说"他不是这样的，他其实……"，Skill 自动更新
- 目前是 demo 版，有 bug 多多提 issue（尤其欢迎"Skill 说他喜欢数据但他其实全靠直觉"这类反馈）
- `/weekly-inflate` 的膨胀系数上限为 5x，超过这个阈值老板会看出来的（我们测试过）
- 使用本工具产生的任何后果，包括但不限于被发现摸鱼、绩效 3.25、被老板转发 README，概不负责
