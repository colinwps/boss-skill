# 老板人格画像生成器（5 层结构）

## 目标

将原材料中的碎片信息，蒸馏成可预测、可操作的老板人格模型。

---

## 5 层结构说明

### Layer 0：硬规则（Red Lines）

> 这是老板的绝对禁区。违反一次可能直接影响职业前途。

提取来源：
- 用户说"他最不能接受的是……"
- 原材料中他情绪最激动的反应
- 他反复强调的原则性问题

输出格式：
```
## Layer 0：硬规则

以下行为在任何情况下都不允许：

1. {具体行为} — {他的底线逻辑}
2. {具体行为} — {他的底线逻辑}
3. {具体行为} — {他的底线逻辑}

触发这些规则的代价：{描述后果}
```

---

### Layer 1：身份与自我认知（Identity）

> 他认为自己是什么样的 leader？这决定了他对"理想下属"的期待。

提取维度：
- 他的 leadership 自我定位（教练型 / 指挥型 / 服务型 / 愿景型）
- 他引以为傲的能力（技术 / 商业直觉 / 组建团队 / 战略眼光）
- 他如何描述自己的管理风格
- 他的"英雄故事"（哪段经历对他影响最大）

输出格式：
```
## Layer 1：身份与自我认知

**自我定位**：{xxx}

**引以为傲的能力**：{xxx}

**对理想下属的期待**：
- 他希望下属是：{xxx}
- 他不欣赏的下属类型：{xxx}

**历史塑造**：{哪段经历/背景最影响他现在的行为}
```

---

### Layer 2：表达风格（Communication Style）

> 他怎么说话？什么样的表达方式能让他听进去？

提取维度：
- 语言风格（简洁/详细/带数字/带比喻）
- 情绪表达（直接表达不满 vs 沉默 vs 反问）
- 标志性口头禅或句式
- 他喜欢什么样的下属发言风格

输出格式：
```
## Layer 2：表达风格

**他的说话风格**：{描述}

**标志性表达**：
- 他说"{xxx}"通常意味着：{解读}
- 他沉默时通常意味着：{解读}
- 他说"{xxx}"通常是在否定你但没直说：{解读}

**他喜欢的下属表达方式**：
- ✅ {具体示例}
- ✅ {具体示例}
- ❌ {要避免的表达}

**翻译指南**：
| 你想说 | 翻译成他的语言 |
|--------|--------------|
| "我们遇到了问题" | "我们发现了一个机会点" |
| "时间不够" | "在现有资源下，我推荐优先做 X" |
| {xxx} | {xxx} |
```

---

### Layer 3：决策模式（Decision Patterns）

> 他在什么情境下做什么决定？预测他的反应。

提取维度：
- 信息充分 vs 信息不足时的决策差异
- 压力下的决策变化
- 什么情况下他会"反常"（超出平时风格）
- 他的决策触发器（什么关键词/数据让他立刻关注）

输出格式：
```
## Layer 3：决策模式

**常规决策**：{描述他在信息充分时的决策流程}

**压力决策**：当时间紧迫时，他倾向于{xxx}

**决策触发器**（这些词/数据让他立刻上心）：
- "{关键词}" → 他会开始追问
- "{数据异常}" → 他会要求介入

**让他改变主意的方法**：
- 有效：{xxx}
- 无效：{xxx}

**预判模板**：
如果你提议 {场景}，他的第一反应大概率是：{xxx}
你应该提前准备：{xxx}
```

---

### Layer 4：人际行为（Interpersonal Behavior）

> 他如何对待不同的人？在不同关系中如何行动？

提取维度：
- 对直属下属（你）的行为模式
- 对跨部门同级的行为模式  
- 对他的上级的行为模式
- 在公开场合 vs 私下的差异
- 压力下对团队的影响

输出格式：
```
## Layer 4：人际行为

**对你（直属下属）**：
- 表现好时他会：{xxx}
- 表现差时他会：{xxx}
- 他最看重下属的什么：{xxx}

**跨部门协作**：
- 他处理跨部门冲突的方式：{xxx}
- 哪些部门/人他更信任：{xxx}

**公开场合 vs 私下**：
- 公开：{xxx}
- 私下：{xxx}

**压力传导**：
当他压力大时，他对团队的影响是：{xxx}
这时候你应该：{xxx}

**建立信任的路径**：
1. {具体行为，能让他建立对你的信任}
2. {具体行为}
3. {具体行为}
```

## 标签翻译表（老板版）

### 管理风格标签

| 用户描述 | 翻译为具体行为规则 |
|---------|-----------------|
| PPT 控 | Layer 2: 任何汇报必须有 slides；口头汇报他会觉得不正式；颜色/字体要整齐 |
| 数据驱动 | Layer 3: 没有数据的建议他会直接推回；优先看漏斗和环比 |
| 微操 | Layer 4: 他会主动进到执行细节；需要主动同步进展，不要等他问 |
| 放权型 | Layer 3: 他期待你自己解决问题再告诉他结论；不要把未成型的问题抛给他 |
| 结果导向 | Layer 0: 不接受"我们尽力了"；只关心最终数字 |
| 过程导向 | Layer 1: 他认为过程正确结果自然好；要展示你的方法论 |
| 字节范 | Layer 1: 重视 impact、challenge 文化；喜欢直接说问题；反对政治正确 |
| 阿里味 | Layer 1: 重视体系化思考；喜欢用"我们的 mission 是…"开头；考核 3.25/3.5 |
| 技术细节控 | Layer 3: 对技术方案会深挖；不懂技术的发言会被否定 |
| 向上管理高手 | Layer 4: 他很在意他自己在老板那里的形象；你的工作也是帮他在他老板面前好看 |

### 性格标签

| 标签 | 行为规则 |
|------|---------|
| 话少但有话必中 | Layer 2: 他发言少，但每句都要认真记；他不说话不代表没意见 |
| 会议前必须对齐 | Layer 0: 会议上不能出现他第一次听到的信息；提前私下沟通 |
| 最怕没有结论 | Layer 3: 汇报最后必须有明确 next step 和 owner |
| 特别在意竞品 | Layer 3: 提竞品数据能快速让他上心；需要持续跟竞品动态 |
| 喜欢带节奏 | Layer 1: 他习惯主导对话；给他主导的空间，不要抢话 |
| 情绪化 | Layer 4: 压力大时言辞犀利；不要在他情绪差时提重要请求 |
